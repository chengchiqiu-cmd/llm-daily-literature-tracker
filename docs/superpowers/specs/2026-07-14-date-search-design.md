# 日期选择与全站论文搜索设计

## 目标

在现有静态 GitHub Pages 文献归档中：

- 将单一日期下拉框拆为年、月、日三个联动选择框。
- 默认选择最新可用报告日期，该日期即当前归档中的最近检索日。
- 支持跨全部已生效报告搜索论文标题关键词或作者姓名。
- 点击搜索结果后打开对应日报，并准确定位到文章正文。

## 数据与生成流程

网站继续采用纯静态生成，不增加后端或外部依赖。构建器读取每份合格报告时，除现有报告元数据外，还从 `article.paper` 中提取：

- 报告日期与文件名；
- 文章块已有的稳定 `id`；
- `h2.paper-title` 中的论文标题；
- `p.paper-meta` 中的作者及单位、发表记录文本。

生成器把这些字段作为精简搜索索引嵌入首页。索引不复制摘要、公式或其他报告正文。

## 日期交互

首页显示年、月、日三个原生选择框。可选值仅来自实际存在的报告：

1. 年份变化后，月份只显示该年份有报告的月份。
2. 月份变化后，日期只显示该年月有报告的日期。
3. 页面首次加载时，三个控件默认选择最新可用报告日期。
4. 用户完成有效日期选择后，打开该日论文分布弹窗。
5. 若归档为空，三个控件和查看按钮保持禁用状态。

## 搜索交互

搜索覆盖全部已生效报告，不受当前日期选择影响。匹配规则为：

- 对论文标题和作者行进行大小写不敏感的包含匹配；
- 支持中文、英文、连字符及姓名空格；
- 去除查询首尾空格；
- 空查询不显示结果；
- 结果按报告日期从新到旧，并保持日报中的文章顺序。

每条结果显示论文题目、作者和报告日期。点击结果打开：

`报告文件名.html#文章稳定ID`

发布时向报告页注入轻量 `:target` 样式，使目标文章获得清晰但克制的高亮，帮助读者确认定位。

## 可访问性与响应式

- 每个日期选择框使用明确的可访问名称。
- 搜索使用 `type="search"`、可见标签和结果状态区域。
- 搜索结果可用键盘访问。
- 桌面端日期控件横向排列；移动端在不压缩文字的前提下换行。
- 不改变报告正文内容、文章排序、Fasheng Xu 聊天通知规则或期刊白名单规则。

## 测试与发布

1. 单元测试验证文章索引提取、三段日期控件、默认日期和搜索索引输出。
2. 同步测试验证发布报告仍只包含生效后的文件，并保留文章锚点。
3. 浏览器测试验证日期联动、默认值、标题搜索、作者搜索、无结果状态及锚点跳转。
4. 在桌面和移动视口检查溢出、弹窗与搜索结果布局。
5. 测试通过后提交发布目录，推送 `main`，等待 GitHub Pages 工作流成功。
