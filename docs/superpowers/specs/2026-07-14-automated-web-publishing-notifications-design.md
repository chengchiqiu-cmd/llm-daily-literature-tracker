# 每日文献报告自动发布与通知设计

## 目标

每日文献追踪完成后，不再把完整“文章信息”或日报正文重复发送到聊天。自动化应把新报告直接加入 **Analytical Models & AI Research Tracker**，等待 GitHub Pages 部署成功，再发送一条简短发布回执。

Fasheng Xu 专项追踪保持独立：只有其作者主页或 SSRN 出现可核实的新增或更新时，才额外发送一条专项提醒。网站与日报中不展示该专项追踪状态。

## 发布事务

一次日报更新只有在以下步骤全部成功后才算完成：

1. 生成符合现行筛选规则的日报 HTML 与结构化元数据；0 篇结果也生成空报告。
2. 重建归档首页、日期分布和跨日期论文搜索索引。
3. 运行内容与站点自动化测试，包括期刊白名单、内部标签、链接和元数据一致性检查。
4. 将首页及合规日报同步到 GitHub Pages 仓库的 `site/` 目录。
5. 仅提交本次报告和站点生成结果，推送 `main` 分支。
6. 等待对应 GitHub Pages workflow 成功完成，并核对公开入口仍可访问。
7. 记录本次已发布的报告日期、检索窗口和提交，以支持补检与防重复。

任何一步失败都不得发送“已更新”回执。上一个成功部署的站点版本继续作为公开版本。

## 聊天通知

### 发布成功回执

每次成功部署后发送一条低密度回执，只包含：

- 报告日期或合并补发日期范围；
- 检索窗口，明确标注北京时间 / Asia-Shanghai；
- 新增或更新论文总数；
- 公开网站链接。

回执不重复列文章题目、作者、摘要或详细分类；完整内容统一在网页中查看。空报告照常回执，并明确写“新增/更新 0 篇”。

### Fasheng Xu 独立提醒

仅当作者主页或 SSRN 有可核实新增或更新时发送，且与普通发布回执分开。提醒包含：

- 论文题目；
- 作者主页或 SSRN 触发来源；
- 可核实的 posted、revised 或 updated 日期；
- 论文链接；
- 一句话说明新增或修改内容。

未发现更新时不发送 Fasheng Xu 状态，也不在发布回执中写“未发现更新”。

### 发布失败告警

若报告生成、校验、推送或 Pages 部署失败，发送一条失败告警，包含失败阶段、报告日期或窗口、可操作的错误摘要。告警不得使用“已上线”“已更新”等成功措辞。

## 防重复与补检

- 同一报告日期和检索窗口若已成功部署，不重复提交、不重复触发 Pages，也不重复发送回执。
- 内容发生实质变化时允许重新发布同一日期，并在回执中标明“报告已修订”。
- 多日漏检时生成合并补发报告，完成一次部署后只发送一次发布回执。
- Fasheng Xu 的同一版本日期只提醒一次；后续新的 revised 或 updated 日期可再次提醒。

## 组件边界

### 日报生成器

负责检索、筛选、去重、报告内容及元数据，不负责声称公开部署成功。

### 站点构建器

继续由 `scripts/build_literature_portal.py` 读取合规日报，生成首页和搜索索引，并同步 GitHub Pages 的 `site/` 目录。

### 发布编排器

新增一个单一命令入口，依次执行构建、校验、Git 变更检查、提交、推送、Pages workflow 等待与公开入口核验。该入口输出机器可读的最终状态，供 heartbeat 决定发送成功回执、专项提醒或失败告警。

### GitHub Pages workflow

继续只负责部署仓库中的 `site/` 静态文件。发布编排器通过提交对应的 workflow run 判断部署结果，避免把其他历史运行误认为本次成功。

## 错误处理

- 没有符合条件的论文：生成并发布空报告，不视为错误。
- 生成内容未通过合规检查：停止，不提交或推送。
- 工作区存在无关变更：只处理明确的生成文件；无法隔离时停止并告警。
- 推送失败：保留本地生成结果，告警并等待下次补发。
- Pages workflow 失败或超时：不发送成功回执，报告失败阶段和 workflow 链接。
- 公开入口暂时不可达：在有限重试后告警，不无限等待。

## 验证标准

1. 普通日报正文不再出现在聊天中。
2. GitHub Pages 部署成功后才发送发布回执。
3. 发布回执包含日期、北京时间检索窗口、论文数量和公开链接。
4. 0 篇报告可以正常上线并收到回执。
5. Fasheng Xu 无更新时没有专项消息，有更新时收到独立消息且不进入网站。
6. 构建、测试、推送或部署失败时只发送失败告警。
7. 相同日期和窗口的重复 heartbeat 不产生重复提交或重复通知。
8. 合并补发只部署一次、通知一次，并覆盖完整缺失窗口。
9. 站点仍只公开 2026-07-14 及之后、符合现行规则的报告。
